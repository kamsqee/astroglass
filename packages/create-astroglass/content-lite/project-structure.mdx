# Project Structure

A high-level view of how this project is organized.

│   │   │   └── footer/        # Theme-specific footers
│   │   ├── sections/

│   │   │   ├── contact/       # Contact sections per theme
│   │   │   ├── cta/           # CTA sections per theme

│   │   │   ├── features/      # Feature sections per theme
│   │   │   ├── footer/        # Footer sections per theme
│   │   │   ├── hero/          # Hero sections per theme
│   │   │   ├── portfolio/     # Portfolio sections per theme
│   │   │   ├── pricing/       # Pricing sections per theme

## Key Concepts

### Themes
Each theme has its own set of section components, header, footer, and CSS. Themes are registered in `themePresets.ts` and their files are listed in `themeRegistry.ts`.

### Palettes
Color palettes are defined in `palettes.ts` and their CSS variables live in `src/styles/palettes/`. Users can switch between palettes at runtime.

### Sections
Landing page sections (Hero, About, Features, etc.) are theme-specific components. The order is controlled by `themePresets.ts`.
